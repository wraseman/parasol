Partition data into k clusters in which each data element belongs to the cluster with the nearest mean.
Parameters:
Name Type Attributes Default Description
k number number of clusters
chartIDs Object charts that will display cluster colors
palette Object <optional>
 d3.schemeCategory10 d3 categorical color palette mapping cluster ids to color see https://github.com/d3/d3-scale-chromatic#categorical for complete list
vars Object <optional>
 null variables used for clustering. NOTE: var data must be numeric
std boolean <optional>
 true convert values to zscores to obtain unbiased clusters
options object <optional>
 Option object
Properties
Name Type Attributes Default Description
maxIterations number <optional>
 100 Maximum of iterations allowed
tolerance number <optional>
 1e-6 Error tolerance
withIterations boolean <optional>
 false Store clusters and centroids for each iteration
distanceFunction function <optional>
 squaredDistance Distance function to use between the points (from ml-distance-euclidean)
seed number <optional>
 Seed for random initialization.
initialization string | Array.<Array.<number>> <optional>
 'kmeans++' K centers in format [x,y,z,...] or a method for initialize the data: * You can either specify your custom start centroids, or select one of the following initialization method: * `'kmeans++'` will use the kmeans++ method as described by http://ilpubs.stanford.edu:8090/778/1/2006-13.pdf * `'random'` will choose K random different values. * `'mostDistant'` will choose the more distant points to a first random pick
